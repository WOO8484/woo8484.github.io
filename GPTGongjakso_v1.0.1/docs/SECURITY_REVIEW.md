# SECURITY_REVIEW — v1.0.1

## 네트워크와 계정
- `INTERNET` 권한을 선언하지 않습니다.
- 네이버 로그인 정보, API 키, 비밀번호를 저장하거나 입력하지 않습니다.
- release 서명 정보는 환경변수/GitHub Secrets로만 주입합니다.

## 접근성 최소 범위
- 접근성 대상은 `com.nhn.android.blog` 한 개입니다.
- 네이버 일반 앱, 브라우저, 기타 앱은 관찰·조작 대상에서 제외합니다.
- 발행·등록·완료·게시·공개·저장·임시저장·예약 계열 버튼은 자동 클릭 금지 목록으로 차단합니다.
- 허용 패키지가 아니면 자동 입력을 중단하고 클립보드 fallback으로 전환합니다.

## 앱 실행 범위
- `네이버 블로그 앱 열기`는 `com.nhn.android.blog`만 실행합니다.
- 네이버 일반 앱 및 모바일 블로그 브라우저 fallback은 제거했습니다.
- 미설치 시 Play 스토어 설치 화면만 안내합니다.

## 저장과 ZIP 보안
- `allowBackup=false`와 데이터 추출 제외 설정을 유지합니다.
- 이미지 저장은 MediaStore scoped storage를 사용합니다.
- ZIP 크기·엔트리 수·압축률·경로 탈출·제어문자·중복 basename 방어를 유지합니다.

## 남은 검증
- 실제 APK 빌드와 단위 테스트 통과 여부
- 실제 기기에서 접근성 입력과 Play Protect 동작
- 네이버 블로그 앱 버전별 화면 호환성
