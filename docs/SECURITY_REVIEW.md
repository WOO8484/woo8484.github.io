# SECURITY_REVIEW — v1.0.0

v0.1.1 의 보안 설계를 유지하며, 추가 기능(해시/이력 DB/공유 수신)에서도 데이터 외부 유출 경로를 만들지 않았습니다.

## 1. 네트워크 격리 (핵심)
- **INTERNET 권한을 선언하지 않습니다.** 앱 프로세스는 어떤 서버로도 소켓 연결/전송을 할 수 없습니다.
- 글·이미지·태그·계정정보·이력의 외부 전송 경로가 원천적으로 존재하지 않습니다.
- 네이버 열기는 다른 앱(브라우저/네이버앱)에 Intent 를 넘기는 방식이라 INTERNET 이 불필요합니다.
- 결과적으로 원격 자동 업데이트/원격 선택자 다운로드도 제공하지 않습니다(수동 파일 교체만).

## 2. 계정/비밀 정보 미취급
- 네이버 로그인 정보를 저장/입력/요청하지 않습니다. 로그인은 사용자가 직접 합니다.
- 소스 어디에도 API 키/시크릿/비밀번호가 없습니다.
- release 서명키/비밀번호는 소스에 포함하지 않고 **환경변수(GitHub Secrets)로만** 주입합니다.

## 3. 접근성 서비스 최소 권한
- 조작 대상 패키지를 `com.nhn.android.blog`, `com.nhn.android.search` **2개로만 제한**(accessibility_service_config `packageNames`).
- 좌표 제스처 미사용(`canPerformGestures=false`), 단일 활성 창만 사용.
- **발행/등록/완료/게시/공개/저장/임시저장/예약/확인** 계열 버튼은 클릭 금지 목록으로 차단.
- 자동 조작 전 현재 화면이 허용된 네이버 패키지인지 매번 확인. 아니면 조작하지 않고 클립보드 fallback.
  → 앱 전환/전화 수신으로 다른 앱이 전면에 오면 자동 입력이 즉시 중단됩니다(READ_PHONE_STATE 등 민감 권한 불필요).

## 4. 클립보드 취급(조건부 정리)
- 자동 입력 실패 시 클립보드 복사로 대체하되, **앱이 복사한 값이 현재 클립보드와 동일할 때만** 지웁니다.
- 사용자가 그 사이 다른 것을 복사했거나 클립보드를 읽지 못하면 지우지 않습니다(데이터 보존/안전).

## 5. 저장/백업 격리
- `allowBackup=false` + `data_extraction_rules` 로 root/file/database/sharedpref/external 을 클라우드 백업·기기이전에서 모두 제외.
- 이력 DB/로그/설정은 앱 내부에만 저장. 로그에는 태그/본문 내용을 남기지 않음.
- 이미지 저장은 MediaStore(scoped storage) 사용 → WRITE_EXTERNAL_STORAGE 등 광범위 권한 불필요.

## 6. ZIP 파싱 방어 (자원 제한, v0.1.1 이식)
- 압축폭탄(과대 용량/고압축률), 다량 엔트리, 경로 탈출(basename 만 사용), 제어문자 파일명, 중복 basename 을 방어.
- 손상/비정상 ZIP 은 앱을 죽이지 않고 사용자 메시지 예외로 안전 중단.
- SHA-256 계산은 스트림이 소비한 바이트에 대해 수행(추가 위험 없음).

## 7. 권한 목록(최소)
`SYSTEM_ALERT_WINDOW`(플로팅), `FOREGROUND_SERVICE`(+`SPECIAL_USE`), `POST_NOTIFICATIONS`(알림). 그 외 저장/전화/위치/연락처 권한 없음.

## 8. 잔여 위험 / 사용자 책임
- 접근성 서비스는 강력한 권한이므로, 사용자가 신뢰하는 화면(네이버 글쓰기)에서만 사용해야 합니다. 본 앱은 대상 패키지를 제한해 위험을 낮췄습니다.
- 발행은 항상 사용자가 직접 확인 후 눌러야 합니다. 앱은 발행을 대신 누르지 않습니다.
- Google Play Protect 통과 및 실기기 동작은 **미검증**(실기기 검증 필요, BUILD_STATUS 참고).
