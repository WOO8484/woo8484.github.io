# BUILD_STATUS — 완료/기초/미검증 구분 (정직 보고)

이 문서는 "실패한 작업을 성공처럼, 미검증을 검증처럼" 보고하지 않기 위한 정직한 상태표입니다.
읽는 사람이 오해하지 않도록, 각 항목을 아래 3단계로 구분합니다.

- ✅ **검증됨**: 실제로 컴파일·실행하여 통과를 확인함.
- 🟩 **구현 완료(빌드 검증 예정)**: 코드가 완성되었으나, 이 작성 환경에 Android SDK 가 없어 컴파일은 GitHub Actions 에서 검증해야 함.
- 🟡 **기초 구현**: 동작하는 최소 구현. 지시서의 취지는 반영했으나 UI/세부 정책까지 완성하지는 않음.
- ⛔ **미검증**: 실제 기기가 있어야만 확인 가능. 현재 확인하지 않음.

## 2. ZIP 처리 — 🟩 구현 완료(빌드 검증 예정)
- v0.1.1 의 보안 로직(50MB/100MB/25MB/2MB/50엔트리/10이미지/압축률100, 경로탈출·제어문자·중복 방어, 손상 ZIP 안전예외)을 **그대로 이식**.
- SHA-256(ZIP 원본/본문) + metadata 2.1 파싱을 최소 추가(기존 판정 흐름 불변).
- ZIP 공유/열기 인텐트(ACTION_SEND/VIEW) 추가.
- 보안 제한 자체의 로직은 v0.1.1 에서 이미 JUnit 검증되었고, 본 프로젝트에도 동일 테스트 포함.

## 3. 자동 입력 보조 — 🟩 구현 완료(빌드 검증 예정) / ⛔ 실동작 미검증
- 접근성 서비스의 안전 로직(네이버 2개 패키지 제한, 금지단어 클릭 방지, 클립보드 fallback, 태그 1개씩 IME_ENTER)을 **그대로 이식**.
- 단계별 타임아웃(태그 5s 등) + 상태머신 연동 추가.
- ⛔ 접근성 실동작(네이버 실제 화면에서의 입력 성공)은 **실기기에서만 확인 가능** → 미검증.

- BoardMatcher·상태머신은 ✅ 순수 로직 검증 완료. 9개 기본 프로필 내장.

## 7. 주변 모듈 — 🟡 기초 구현
지시서의 방대한 주변 기능 중, 다음은 **동작하는 기초 구현** 수준입니다(정직히 표기).
- `selector/SelectorRules` (지시서 17): 내장 기본값 + 파일 교체식 수동 갱신. 원격 자동 업데이트는 INTERNET 미보유로 미제공.
- `instruction/InstructionManager` (지시서 22): 지시문 버전/본문 보관 및 대조. 편집 UI 는 미완.
- `diagnostics/DiagnosticsExporter` (지시서 24): 상태/로그/이력 요약 JSON·CSV 내보내기(다운로드 폴더).
- 다음 기능은 **이번 범위에서 전용 UI/전체 구현을 하지 않았습니다**(임의 확장 금지 원칙 준수): 저장소 100GB 정리 자동화(지시서 20), 백업/복원 전체 플로우(지시서 21), 앱 업데이트(수동 APK) 플로우(지시서 25)의 UI. 데이터 구조/원칙은 문서에 반영.

## 8. 빌드/서명 — 🟩 (CI에서 검증)
- Gradle Wrapper(gradlew, gradle-wrapper.jar 8.9) 는 v0.1.1 에서 검증된 것을 재사용.
- GitHub Actions: `clean test assembleDebug` 자동 실행 + debug APK/테스트결과/빌드로그 아티팩트.
- release 서명: Secrets(RELEASE_KEYSTORE_BASE64 등)로만 주입, 소스에 키/비밀번호 없음.
